Introduction to Rubber Ducky

The Rubber Ducky is a microcontroller USB that acts like a keyboard when plugged into a computer and executes a set of instructions that you load into it. This does not mean that you can put malicious files and have the victim computer run them. When creating a Rubber Ducky executable, you have to pretend you're at your victim's computer typing. Will you open a run prompt and use the command line or open a powershell prompt? That's all up to you.

Writing our first DuckyScript file

The heart of Rubber Ducky is the simple scripting language used to create payloads.

Here's the list of commands:

  • REM
  • DEFAULT_DELAY
  • DELAY
  • STRING
  • GUI
  • MENU
  • SHIFT
  • ALT
  • CONTROL
  • LEFTARROW
  • RIGHTARROW
  • DOWNARROW
  • UPARROW
  • BREAK
  • CAPSLOCK
  • DELETE
  • END
  • ESCAPE
  • HOME
  • INSERT
  • NUMLOCK
  • PAGEUP
  • PAGEDOWN
  • PRINTSCREEN
  • SCROLLOCK
  • SPACE
  • TAB
For more information on these commands, please go here

Let's create a simple Rubber Ducky payload that opens notepad and types in "Hello students!" Open your favorite text editor and type in the following:

REM This only works against a Windows victim!
GUI r
DELAY 100
STRING notepad.exe
ENTER
DELAY 100
STRING Hello students!

Save the file as inject.txt and Congratulations! You have just created your first DuckyScript! Find how to encode and load it onto your Rubber Ducky in the next section.
